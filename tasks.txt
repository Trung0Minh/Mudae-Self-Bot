Build a Discord self-bot tool that automates interaction with the Mudae bot based on fixed hourly intervals.

Core requirements:

1. Time-based rolling system:

* The bot must detect real-world clock time and divide it into hourly intervals (e.g., 09:00–10:00, 10:00–11:00, etc.).
* At the exact start of each new hour (HH:00:00), the bot should automatically trigger a roll sequence.
* The system must NOT use a simple 1-hour cooldown from last roll; instead, it must align strictly with real clock hour boundaries.
* It should support the “edge case” where a user rolls at HH:59:xx and then immediately rolls again at (HH+1):00:00.

2. Rolling behavior:

* At each interval, send the appropriate Mudae command (e.g., $wa or configured command) exactly once per interval.
* Each roll sequence should execute the equivalent of 10 rolls (depending on server settings).
* Add a small randomized delay (e.g., 0.5–2 seconds) before sending commands to simulate human behavior.

3. Claim automation:

* Monitor messages from the Mudae bot in real time.
* Parse responses to detect valid claimable characters.
* If a character meets predefined conditions (e.g., not already claimed, matches wishlist, or above a certain value), automatically send the claim command.
* Ensure claim is only attempted if claim cooldown is available.

4. Anti-detection behavior:

* Introduce random delays between actions.
* Avoid perfectly consistent timing patterns.
* Optionally add slight delay after the hour mark (e.g., 1–3 seconds) before rolling.

5. Configuration:

* Allow user to configure:

  * Target Discord channel ID
  * Roll command (e.g., $wa, $ha)
  * Claim conditions (wishlist, value threshold, etc.)
  * Timezone handling
* Include logging for actions (rolls, claims, errors).

6. Technical constraints:

* Use a Discord self-bot approach (user token-based).
* Maintain a persistent connection to Discord (WebSocket or library-based).
* Ensure the bot can run continuously and handle reconnections.

7. Safety note:

* This tool is for experimental or personal testing purposes only (e.g., using a secondary account).
